04 · Landing High fidelity

Landing — “Split Duo”

A 50/50 split entry that visually states FuelUp's two equal sides — donor (cyan) and creator (violet) — echoing the "AI Equality" theme. Interactive: hover a half to preview-expand, click to enter (Expand & Enter transition), click the center FuelUp pill to reset. Try it below.

Notes

  • Inline SVG line icons (give-heart / idea-bulb), not emoji.
  • Glassmorphism logo pill floating over a glowing divider.
  • Soft radial glows — cyan top, violet bottom — for the "two equal sides" metaphor.
  • Transition: Expand & Enter (selected half fills, loading state).
  • Built with pure CSS (hover + hidden-radio state machine) — no JS, smooth on screen recording.
  • Real 393pt width with Dynamic Island & home-indicator safe areas.

06 · Screens

Screen breakdown

Six screens beyond Landing, grouped by track. All share the tokens and components above. Each card notes its in-app navigation — only down-drill screens need a back control; track entry points don't.

Donor track

Browse Structure defined

  • Header with FuelUp logo + short tagline.
  • Vertical list of project cards: name, one-line pitch, progress bar ($ raised / goal), token estimate.
  • Seeded mock data; tap a card → Detail.
⌂Entry of donor track — no back. Forward: tap a project card → Detail + Donate.

Detail + Donate Structure defined

  • Project title, description, intended AI use case.
  • Fundraising status + progress bar (emerald when funded).
  • Amount chips ($10 / $30 / $50 / custom) → live token estimate ("$30 → ~1.5M GPT-4o tokens").
  • Primary CTA "Fund this idea" → mocked payment.
←Back to Browse (top-left). Forward: "Fund this idea" → Donation Success.

Donation Success Structure defined

  • Success animation (e.g. burst / check).
  • Impact line: "$30 → ~1.5M GPT-4o tokens for Alex".
  • Updated project status; "Back to projects".
  • Demo idea: animate value flowing toward the creator.
✓Terminal screen — no back arrow. A forward CTA "Back to projects" returns to Browse.
Idea Owner track

My Ideas Structure defined

  • List of the creator's ideas with per-idea status + balance.
  • "+ Post new idea" → creation form.
  • Select an idea → its Dashboard.
⌂Entry of idea-owner track — no back. Forward: select an idea → Idea Dashboard, or "+ Post new idea" → New Idea.

Idea Dashboard Structure defined

  • Available balance (emerald), incoming donations.
  • Platform-issued API key with copy affordance.
  • "Run live API call" → real response shown (the demo's payoff moment).
←Back to My Ideas (top-left) — this is a per-idea drill-in.

Post New Idea Structure defined

  • Form: name, pitch, goal amount, intended AI use case.
  • Submit → new idea appears in My Ideas.
←Cancel / Back to My Ideas (top-left, form screen). On Submit: return to My Ideas.

07 · Platform

Mobile & safe-area spec

Designed for modern phones (iPhone 17 Pro class). Build flexible, not pixel-locked.

  • Base logical width 390–402pt (Pro class); Pro Max ≈ 430pt. Layouts flex across this range.
  • Reference height ≈ 852–874pt.
  • Respect safe areas with env(safe-area-inset-*) + viewport-fit=cover: top Dynamic Island ≈ 59pt, bottom home indicator ≈ 34pt.
  • Keep primary actions out of the safe-area zones; use relative units, flex, and min/max sizing.
